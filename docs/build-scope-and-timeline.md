# 作る範囲と期間の目安

「あなた（ユーザー）＋ 私（AI）」で、**どこまでをいつ頃つくれるか**の目安。

---

## 1. MVP（まず出したいもの）の範囲

| 項目 | 内容 |
|------|------|
| **Webアプリ** | 1つのルームで、自分＋複数AIが同じスレッドで会話できるUI |
| **認証** | 会員登録・ログイン（メール or Googleなど） |
| **チャット** | メッセージ一覧・送信・複数AIの返信表示（誰が話したか分かる） |
| **AI** | まずは1〜2プロバイダ（例：Gemini）から。同じルームに複数「キャラ」として複数回呼ぶ or 複数モデル |
| **課金** | シンプルな月額サブスク（1980 or 3000円）、決済はStripe等 |
| **利用制限** | 月あたりメッセージ数などの上限（課金と連動） |

**最初からは入れない（後回し）**
- 複数プロバイダ（Claude・GPT等）の同時対応
- アプリ（ネイティブ）
- 高度なマーケティング用ランディング

---

## 2. 役割のイメージ

| 役割 | あなた | 私（AI） |
|------|--------|----------|
| 仕様・優先順位 | 決める | 提案・整理 |
| デザイン・文言 | 希望を伝える・OK出す | 案を出す |
| 実装（コード） | 環境用意・デプロイ・軽い修正 | 設計・コード作成 |
| テスト・確認 | 実際に触ってフィードバック | 不具合修正 |
| マーケティング | 主担当 | 文案・構成の提案 |

---

## 3. 期間の目安

**「あなたの使える時間」**でかなり変わる。

### パターンA：週に5〜10時間ほど使える（仕事と並行）

| フェーズ | 内容 | 目安 |
|----------|------|------|
| 環境・設計 | 技術選定、リポジトリ、DB・認証方針 | 1週間 |
| コア実装 | 認証・ルーム・チャットUI・1〜2体のAI応答 | 2〜3週間 |
| 課金・制限 | Stripe連携、プラン・上限 | 1〜2週間 |
| 調整・公開 | バグ修正、デプロイ、ランディング | 1週間 |

→ **合計 約5〜7週間（1.5〜2ヶ月）** で「課金できるMVP」まで。

### パターンB：短期集中（週20時間以上かけられる）

→ 同じ範囲を **約2〜4週間** で詰められる想定。

### パターンC：まず「課金なし」で形だけ

- 認証＋1ルーム＋複数AIの会話だけ。課金は後付け。
→ **約2〜3週間**（パターンAベース）で動くところまで。

---

## 4. 前提・注意

- **あなたがコードを書けるか**で変わる。書けるなら「私がコード、あなたがレビュー・手元で動かす」で進みが早い。書かないなら「仕様を固めてから実装」の塊になる。
- **デザインをどこまでやるか**でも変わる。まずはシンプルなUIでよく、あとからブラッシュアップも可能。
- マーケティングは**作ってから**でも間に合う。まずは「作って、自分で使ってみて、知り合いに使ってもらう」から始めると現実的。

---

## 5. 早いパターン：デザインを凝らさない場合

**「動くもの」だけなら、かなり短くできる。**

| 目標 | 目安 |
|------|------|
| **1日** | 認証なし・課金なし。1画面で「ユーザーが送信 → 複数AI（例：Gemini 2体や別モデル）が返信」が動くプロトタイプ。Next.js + API で実装すれば、集中すれば **1日で形にできる**。 |
| **数日** | 上に「誰が話したか表示」「履歴を保持」を足しただけのミニマム。 |
| **1〜2週間** | 認証・ルーム・課金を足した「売れる形」のMVP。 |

以前の「1.5〜2ヶ月」は**認証・課金・複数ルーム・デプロイ・調整まで含めた「リリースできる製品」**を想定していた。デザインを凝らさず「まず動かす」なら、**1日で核の体験（複数AIとの雑談）は作れる**。

---

## 6. 一言まとめ

- **「複数AIが返すチャットが動く」だけなら、凝らなければ 1日で作れる。**
- 「売れる形（課金・認証・制限）」まで足すと、数週間〜1.5ヶ月のイメージ。

次の一歩：**今日〜明日で「1画面・複数AIが返す」プロトタイプから作る**のが現実的。
